# PatentClaw - 专利龙虾

一个基于 [OpenClaw](https://github.com/nicepkg/openclaw) 的 AI Skill，专为**中国发明专利申请文件**自动生成而设计。

采用**多 Agent 编排架构**，通过 4 个专业子 Agent 并行协作，覆盖从学术论文到专利四件套的全流程，自动完成术语一致性审校，最终导出符合国家知识产权局（CNIPA）排版要求的 `.docx` 文件。

## 架构

```
Phase 1: 编排器（交互）
  用户提供论文/交底书 → 解析 → 确认创新点 → 收集申请人信息
  │
Phase 2: 3 个子 Agent 并行
  ├── 权利要求策略师 ──→ 3 套策略（宽/稳/窄）
  ├── 说明书撰写师  ──→ 说明书 + 摘要
  └── 现有技术检索员 ──→ 查新报告 + 建议
  │
Phase 3: 编排器（交互）
  展示查新报告 + 3 套策略 → 用户选择 → 组装四件套
  │
Phase 4: 1 个子 Agent
  └── 审校修复师 ───→ 自动检查 → 修复 → 重检（最多3轮）
  │
Phase 5: 编排器（导出）
  展示最终文档 → 导出 .docx → 提醒用户找代理人审核
```

### 为什么用多 Agent？

| 维度 | 单 Agent（v1） | 多 Agent（v2） |
|------|---------------|---------------|
| 权利要求策略 | 生成 1 套 | 并行生成 3 套供选择 |
| 生成效率 | 串行生成 | 权利要求和说明书并行 |
| 查新能力 | 无 | 自动检索现有技术 |
| 术语审校 | 生成后手动检查 | 独立 Agent 自动检查+修复 |
| 上下文利用 | 所有工作共享一个窗口 | 各 Agent 独立上下文 |

## 功能特性

### 两种工作模式

**模式一：论文 → 技术交底书**

提供学术论文，PatentClaw 自动完成论文解析、创新点识别、上位化改写、生成技术交底书。

**模式二：交底书 → 专利四件套**（多 Agent 编排）

提供技术交底书，PatentClaw 编排 4 个子 Agent 并行生成：
- **权利要求策略师** — 生成宽保护/稳妥/窄保护 3 套策略，用户对比后选择
- **说明书撰写师** — 并行撰写说明书和摘要
- **现有技术检索员** — 搜索相关专利和论文，评估新颖性，建议策略
- **审校修复师** — 自动运行一致性检查，修复术语不一致等问题

### 自动一致性检查

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 术语一致性 | error | 权利要求书与说明书用词必须完全一致 |
| 从属权利要求重叠 | error | 附加特征不得与独立权利要求重复 |
| 引用编号合法性 | error | 从属权利要求不得前向引用 |
| 说明书覆盖度 | warning | 权利要求的特征须在说明书中充分描述 |
| 独立权利要求宽度 | warning | 检测限定过多或含具体数值 |
| 摘要字数 | error | 超过 300 字自动精简 |

### 国知局格式 .docx 导出

- A4 纸张，页边距：上下 25.4mm，左右 25mm
- 标题：小二号黑体加粗居中
- 一级标题：四号黑体加粗
- 正文：小四号宋体，首行缩进 2 字符，1.5 倍行距
- 页脚居中页码

## 安装

### 前置条件

- [OpenClaw](https://github.com/nicepkg/openclaw) 已安装并完成初始化
- Node.js >= 18

### 安装步骤

1. 克隆到 OpenClaw workspace 的 `skills/` 目录：

```bash
cd ~/.openclaw/workspace
mkdir -p skills
cd skills
git clone https://github.com/CalicoCatto/PatentClaw.git patent-claw
```

2. 安装依赖：

```bash
cd patent-claw
npm install
```

3. 在 OpenClaw 对话中输入包含"专利""交底书""四件套"等关键词即可触发。

## 使用方式

### 通过 OpenClaw 对话

```
用户：这篇论文要申请专利 [发送论文]
龙虾：[解析论文 → 识别创新点 → 上位化 → 生成交底书]

用户：继续生成四件套，发明人张三、李四，申请人XX大学...
龙虾：正在并行启动 3 个专业子 Agent 工作...
     ├── 权利要求策略师：生成 3 套策略
     ├── 说明书撰写师：撰写说明书
     └── 现有技术检索员：检索相关专利

龙虾：[展示查新报告 + 3 套策略对比表]
     请选择一套权利要求策略：(1) 宽保护 (2) 稳妥 (3) 窄保护

用户：选 2
龙虾：[审校修复师自动检查修复 → 展示最终文档 → 导出 .docx]
```

### 单独使用工具脚本

```bash
# 一致性检查
node check-consistency.js input.json

# 导出 .docx
node export-docx.js input.json output_dir/
```

## 核心原则

1. **生成的是初稿，不是终稿** — 每次输出提醒用户须经专利代理师审核
2. **绝不捏造身份信息** — 发明人、申请人等必须用户提供
3. **专利思维，不是论文思维** — 抽象概括以最大化保护范围
